## 技术方案（Phase 2 结论）

### 输出结构
```
output/
├── person_001/
│   ├── video_a.mp4          ← 原视频的软链接或副本
│   ├── video_c.avi
│   ├── cover_1.jpg
│   ├── cover_2.jpg
│   └── cover_3.jpg
├── person_002/
│   └── ...
├── unknown/                 ← 无法识别到人脸的视频
└── damaged/                 ← 损坏无法读取的视频
```

### 处理流程（4 步）
1. **视频健康检查** — 用 `ffprobe` 检测，损坏视频移入 `damaged/`
2. **人脸提取** — 均匀抽帧（每 30 秒一帧），提取人脸特征向量
3. **人脸聚类** — 用 DBSCAN 算法将相似人脸自动分组
4. **整理输出** — 创建人员文件夹，放入视频（软链接），生成 3 张封面图

### 核心依赖库
| 库 | 用途 |
|---|---|
| `face_recognition` | 人脸检测 + 特征提取（基于 dlib） |
| `opencv-python` | 视频读帧、图片处理 |
| `scikit-learn` | DBSCAN 聚类算法 |
| `ffmpeg-python` | 视频健康检查 |
| `tqdm` | 进度条显示 |

### 复杂度评估
- 整体难度：**中等**
- 性能预估：300 个视频约需 **1~3 小时**（MacBook Pro CPU）
- 扫描结果会缓存，重跑不重新提取
